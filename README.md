##  iOS代码评审标准
>   代码就像自己的孩子一样，为什么不能不竭尽所能的把它打扮的漂漂亮亮的呢？

按照从高到底的要求可以分为：

1.  能用、无bug
2.  代码注释，代码缩进和空行，方法、类、协议，文件，图片命名，大小写等基本编程规范
3.  代码是否精练（不可增加和减少），程序命名是否优雅
4.  代码逻辑结构和程序的逻辑结构划分是否合理，是否还存在可以优化的空间
5.  代码提交是否符合规范
